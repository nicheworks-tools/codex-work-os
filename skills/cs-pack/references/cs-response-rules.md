# CS Response Rules

## Core rules
- Clarity before detail.
- Preserve evidence that could help the next owner.
- Avoid blame language.
- Do not overstate certainty.
- Make the next step explicit.

## Strong CS output signals
- clear issue statement
- visible user impact
- reproduction clues preserved
- prior actions documented
- concise next-step recommendation

## Weak CS output patterns
- vague "customer has an issue" phrasing
- missing impact
- no reproduction hints
- no record of actions already taken
- defensive or blame-heavy wording

## Compression rule
If the source material is messy, compress toward:
1. what is happening
2. who is affected
3. what evidence exists
4. what has already been tried
5. what should happen next
