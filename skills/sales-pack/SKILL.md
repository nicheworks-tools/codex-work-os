# Sales Pack

## Purpose
Turn messy inbound sales context into clear, sendable, commercially useful outputs.

## Use this pack when
- an inbound inquiry is vague or scattered
- a reply draft is needed quickly
- you need to identify missing information before responding
- a sales-oriented next step needs to be proposed
- notes must be turned into a concise proposal brief

## Do not use this pack when
- the main task is a PM status summary
- the task is a support escalation or customer issue handoff
- the request is a research comparison memo
- the request is executive support rather than prospect communication
- the task is a code implementation

## Inputs
- customer or prospect message
- internal notes
- product or offer constraints
- preferred next step if known
- timeline or commercial sensitivity if known

## Required behavior
1. Separate confirmed facts from assumptions.
2. Surface missing information before overcommitting.
3. Keep tone professional, helpful, and easy to send.
4. Make the next step explicit.
5. Avoid invented pricing, scope, or promises.

## Default output order
1. Goal
2. Confirmed facts
3. Open questions
4. Recommended next step
5. Ready-to-send draft

## Quality standard
A useful output should help the user:
- understand the actual inquiry
- see what is still unclear
- avoid overpromising
- send a clean next-step response with minimal editing
