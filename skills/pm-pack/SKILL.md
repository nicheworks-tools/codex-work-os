# PM Pack

## Purpose
Turn scattered project information into short, decision-friendly PM outputs.

## Use this pack when
- progress updates are spread across Slack, notes, docs, or chats
- a weekly status brief is needed
- blockers and dependencies need to be surfaced
- stakeholders need a quick reality-based snapshot
- decisions or missing owners need to be clarified

## Do not use this pack when
- the request is purely for outward-facing sales communication
- the request is a customer support escalation
- the request is a deep research comparison memo
- the user needs a code change rather than a PM summary

## Inputs
- notes, bullet lists, meeting scraps, or thread excerpts
- target audience if known
- reporting window if known
- known blockers, dependencies, deadlines, and decisions

## Required behavior
1. Separate facts from interpretation.
2. Prefer signal over completeness.
3. Surface blockers and decisions early.
4. State missing context explicitly instead of inventing certainty.
5. End with concrete next actions whenever useful.

## Default output order
1. Goal / scope
2. Current status
3. Progress made
4. Risks / blockers
5. Decisions needed
6. Recommended next actions

## Quality standard
A stakeholder who did not follow the original thread should still understand:
- what happened
- what matters now
- what is blocked
- what should happen next
