# Naming

## General rule

Names should describe operational roles, not aspirational vibes.

Prefer:

- `pm-pack`
- `research-pack`
- `sales-pack`
- `meeting-pack`
- `decision-log`

Avoid:

- `super-pack`
- `business-genius`
- `ultimate-ai`
- `misc-tools`

---

## Why pack naming matters

Clear names reduce:

- installation confusion
- invocation confusion
- overlap between skills
- future maintenance friction
