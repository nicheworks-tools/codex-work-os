# End-to-End Scenarios

## Purpose
This repository is skills-first, but skills should still be understandable in realistic multi-step situations.

## Current scenario
### pilot-rollout
Location:
- `examples/end-to-end/pilot-rollout/`

This scenario shows how one messy situation can move across packs:
1. raw input
2. research memo
3. PM brief
4. executive brief
5. sales reply
6. CS escalation

## Why this matters
Without end-to-end examples, packs can feel disconnected.
This scenario shows that the repository is not a random collection of files.
It is a structured system for turning noisy work into usable outputs.
