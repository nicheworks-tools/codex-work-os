# CS Escalation Flow

## Purpose
Turn a long support case into a concise escalation package.

## Use when
- A case is too long or messy
- Engineering or management needs a quick handoff
- Similar incidents need to be grouped or compared

## Inputs
- Customer messages
- Support notes
- Reproduction steps if any
- Current severity and impact

## Process
1. Summarize the user-facing problem.
2. Identify impact, urgency, and current status.
3. Extract reproduction signals and evidence.
4. List what has already been tried.
5. Write a concise escalation note.

## Required output
1. Case summary
2. User impact
3. Evidence / reproduction
4. Actions already taken
5. Escalation recommendation
6. Ready-to-send escalation note

## Quality bar
- No blame language
- No missing reproduction clues
- The next owner should know exactly what to do next
