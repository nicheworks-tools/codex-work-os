# Codex Work OS 仕様書（日本語）

## 1. 目的

Codex に対して、開発以外を含む反復実務を安定して行わせるためのリポジトリを作る。

対象領域は以下の5つとする。

- 調査・比較
- PM / 進行整理
- 営業返信・案件整理
- 秘書・経営補佐
- CS / サポート整理

---

## 2. プロダクト定義

このリポジトリは、単発のプロンプト集ではなく、Codex 向けの実務運用基盤である。

必須レイヤーは以下とする。

- リポジトリ全体の運用ルール
- pack 単位の `SKILL.md`
- 共有テンプレート
- 共有リファレンス
- install / uninstall 補助
- リポジトリ検証スクリプト
- 例示
- 仕様書

---

## 3. MVP 定義

以下を満たしたら MVP とみなす。

- clone して構造が理解できる
- pack を `~/.agents/skills` に導入できる
- 5 pack すべてに実用的な `SKILL.md` がある
- 共通テンプレートが存在する
- 軽量な検証スクリプトがある
- 5 pack すべてに例がある

---

## 4. Internal RC 定義

以下を満たしたら Internal RC とみなす。

- docs がまとまっている
- 構造に意図があり、雑な寄せ集めに見えない
- examples が利用イメージを十分伝える
- install の流れが現実的である
- shared references により出力のブレが減る
- placeholder 段階を明確に超えている

---

## 5. 公開品質までの差分

公開品質に達するには、さらに以下が必要となる。

- 実際の Codex ワークフローでの検証
- 実使用に基づく導入説明の改善
- CI や追加検証の整備
- より現実的な例と pack の磨き込み
