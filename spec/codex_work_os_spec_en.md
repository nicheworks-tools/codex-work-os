# Codex Work OS Spec (EN)

## 1. Objective

Create a Codex-native repository for recurring non-dev and cross-functional work.

The repository should help Codex produce structured, reusable outputs in five practical domains:

- research
- project management
- sales
- executive assistance
- customer support

---

## 2. Product definition

This repository is a reusable operating system, not a single prompt pack.

Its required layers are:

- repository-wide operating rules
- pack-level skill files
- shared templates
- shared references
- install and uninstall helpers
- repository validation scripts
- examples
- source-of-truth specs

---

## 3. MVP definition

MVP is reached when:

- the repository can be cloned and understood
- packs can be installed into a Codex-readable location
- all five packs have usable `SKILL.md` files
- shared templates exist for common output shapes
- lightweight validation scripts exist
- examples exist for all five packs

---

## 4. Internal RC definition

Internal RC is reached when:

- the docs explain the repo clearly
- the structure feels coherent and intentional
- the examples are strong enough to teach usage
- the install path is practical
- the shared references reduce drift
- the repo is materially beyond placeholder quality

---

## 5. Publication-grade gap

The remaining gap to publication-grade quality includes:

- real-world validation in Codex workflows
- clearer public onboarding based on actual usage
- stronger automated checks or CI
- more realistic examples and pack refinements
